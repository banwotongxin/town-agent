# 赛博小镇 V2 - 图工作流引擎技术文档

## 1. 概述

图工作流引擎是赛博小镇V2的核心调度系统，基于状态图（State Graph）模式实现。它负责管理整个小镇的运行流程，包括角色分发、对话处理、多智能体协作等复杂工作流。系统采用模块化设计，支持灵活的节点组合和条件路由。

## 2. 架构设计

### 2.1 核心组件

```
TownOrchestrator (小镇编排器)
├── AgentGraph (智能体图谱)
│   ├── dispatch_node (分发节点)
│   ├── route_to_agent_node (路由节点)
│   └── should_continue (条件判断)
├── Agent Management (角色管理)
│   ├── addAgent/removeAgent
│   ├── getAgent/listAgents
│   └── getOrCreateAgentGraph
└── State Management (状态管理)
    ├── TownState (小镇状态)
    └── AgentState (智能体状态)
```

### 2.2 工作流程

```
用户输入 → dispatchNode → routeToAgentNode → 返回回复
              ↓                ↓
         选择目标角色      调用AgentGraph处理
```

## 3. 核心类详解

### 3.1 TownOrchestrator类

小镇编排器是整个系统的核心管理类，负责协调所有角色和子系统。

#### 3.1.1 主要属性

- `townName`: 小镇名称
- `agents`: 所有角色的集合（Record<string, BaseAgent>）
- `agentGraphs`: 每个角色的图谱实例（Record<string, AgentGraph>）
- `agentMemories`: 每个角色的记忆系统（Record<string, DualMemorySystem>）
- `skillRegistry`: 技能注册表
- `emotionEngine`: 情感引擎
- `state`: 当前小镇状态（TownState）

#### 3.1.2 TownState接口

定义了小镇在处理用户请求时的完整状态：

```typescript
interface TownState {
  user_input: string;                    // 用户输入的消息
  target_agent_id: string;               // 用户指定的目标角色ID
  selected_agent_id: string;             // 实际选中的角色ID
  agent_response: string;                // 角色的回复内容
  conversation_history: BaseMessage[];   // 完整的对话历史
  should_continue: boolean;              // 是否继续对话流程
}
```

#### 3.1.3 核心方法

##### chat() 方法
处理用户与角色的单轮对话，是最常用的接口。

**工作流程：**
1. 初始化小镇状态
2. 执行dispatchNode确定目标角色
3. 执行routeToAgentNode生成回复
4. 更新对话历史并返回结果

**返回值：**
```typescript
{
  response: string;              // 角色回复
  agent_id: string;              // 角色ID
  agent_name: string;            // 角色名称
  conversation_history: BaseMessage[];  // 更新后的对话历史
}
```

##### multiAgentChat() 方法
支持多个角色围绕一个话题进行多轮对话。

**参数：**
- `topic`: 讨论的话题
- `participantIds`: 参与对话的角色ID数组
- `maxRounds`: 最大对话轮数（默认3）

**返回值：** 聊天记录数组，每条记录包含轮次、角色信息和回复内容。

##### addAgent() 方法
向小镇添加一个新角色，同时为其创建记忆系统。

**参数：**
- `agent`: 要添加的角色实例
- `memoryWindow`: 记忆窗口大小（默认5）

### 3.2 AgentGraph类

智能体图谱负责处理单个角色的对话逻辑，集成记忆、技能、情感等系统。

#### 3.2.1 主要功能

- **消息处理**: 接收用户消息，生成角色回复
- **记忆集成**: 自动加载和保存对话记忆
- **技能匹配**: 根据用户输入匹配相关技能
- **情感计算**: 管理角色间的情感关系
- **工具调用**: 支持外部工具的调用和执行

#### 3.2.2 processMessage() 方法

处理单条消息的核心方法：

```typescript
async processMessage(
  userInput: string,
  conversationHistory: BaseMessage[]
): Promise<{ response: string }>
```

**处理流程：**
1. 从记忆系统获取上下文
2. 格式化消息（添加系统提示、记忆上下文）
3. 调用角色的respond方法生成回复
4. 保存对话到记忆系统
5. 返回回复内容

### 3.3 节点系统

图工作流由多个节点组成，每个节点负责特定的处理任务。

#### 3.3.1 dispatchNode (分发节点)

**功能：** 确定用户要与哪个角色对话。

**逻辑：**
- 如果用户指定了target_agent_id，使用该ID
- 否则，自动选择第一个可用角色
- 设置selected_agent_id字段

#### 3.3.2 routeToAgentNode (路由节点)

**功能：** 将消息路由到选定的角色并获取回复。

**处理步骤：**
1. 检查selected_agent_id是否有效
2. 获取或创建该角色的AgentGraph
3. 调用graph.processMessage处理消息
4. 更新state中的agent_response
5. 将用户消息和AI回复添加到conversation_history

#### 3.3.3 shouldContinue (条件判断)

**功能：** 判断是否继续对话流程。

**返回值：**
- "continue": 继续处理
- "end": 结束流程

## 4. 状态管理

### 4.1 状态流转

```
初始状态 → dispatchNode → routeToAgentNode → 最终状态
    ↓           ↓              ↓                ↓
  empty     确定角色      生成回复        包含完整信息
```

### 4.2 状态持久化

- **短期状态**: 保存在内存中，随请求生命周期存在
- **长期状态**: 通过记忆系统持久化到文件系统或数据库
- **会话状态**: 通过conversation_history在多次请求间传递

## 5. 角色管理

### 5.1 角色生命周期

```
创建 → 添加 → 使用 → 移除 → 清理
```

#### 5.1.1 添加角色

```typescript
orchestrator.addAgent(agent, memoryWindow);
```

**执行操作：**
1. 将角色添加到agents集合
2. 为角色创建DualMemorySystem
3. 初始化记忆窗口大小

#### 5.1.2 移除角色

```typescript
orchestrator.removeAgent(agentId);
```

**清理资源：**
1. 从agents集合删除
2. 删除对应的agentGraph
3. 删除对应的agentMemories

### 5.2 默认角色配置

系统提供8个默认角色配置（DEFAULT_PROFILES）：
- 作家 (writer)
- 医生 (doctor)
- 程序员 (coder)
- 教师 (teacher)
- 艺术家 (artist)
- 工程师 (engineer)
- 科学家 (scientist)
- 商人 (businessman)

使用`createDefaultTown()`函数可以快速创建包含默认角色的小镇。

## 6. 多智能体协作

### 6.1 群聊机制

multiAgentChat方法实现了多角色围绕话题的轮流对话：

```typescript
for (let round = 0; round < maxRounds; round++) {
  for (let i = 0; i < participantIds.length; i++) {
    const agentId = participantIds[i];
    const prevAgentId = i > 0 ? participantIds[i - 1] : "user";
    
    // 获取角色的AgentGraph
    const graph = getOrCreateAgentGraph(agentId, prevAgentId);
    
    // 处理消息（以上一个角色的回复作为输入）
    const result = await graph.processMessage(currentTopic, []);
    
    // 更新话题为当前回复
    currentTopic = result.response;
  }
}
```

### 6.2 应用场景

- **头脑风暴**: 多个角色从不同角度讨论问题
- **角色扮演**: 模拟真实场景中的多方对话
- **协作创作**: 多个角色共同完成创作任务
- **辩论赛**: 不同观点的角色进行辩论

## 7. 扩展性设计

## 8. 性能优化

### 8.1 懒加载策略

- **AgentGraph懒创建**: 只在首次使用时创建
- **记忆系统按需加载**: 从文件加载历史对话时限制token数量
- **技能延迟加载**: 仅在需要时加载相关技能

### 8.2 缓存机制

- **AgentGraph缓存**: 避免重复创建图谱实例
- **记忆缓存**: 短期记忆保留在内存中
- **技能缓存**: 已加载的技能缓存在注册表中

### 8.3 异步处理

所有I/O操作都使用async/await：
- 非阻塞的文件读写
- 并发的数据库操作
- 高效的LLM调用

## 9. 错误处理

### 9.1 异常处理策略

- **角色不存在**: 返回友好的错误消息
- **记忆加载失败**: 使用fallback机制，记录日志
- **LLM调用失败**: 返回默认回复或错误提示

### 9.2 容错机制

- **空值检查**: 对所有可能为空的值进行检查
- **类型验证**: 验证输入参数的类型和格式
- **资源清理**: 确保异常情况下正确清理资源

## 11. 使用示例

### 11.1 基本对话

```typescript
// 创建小镇
const town = await createDefaultTown(4);

// 与特定角色对话
const result = await town.chat(
  "你好，请介绍一下你自己",
  "agent_writer"
);

console.log(result.agent_name + ": " + result.response);
```

### 11.2 多角色讨论

```typescript
// 多个角色讨论一个话题
const records = await town.multiAgentChat(
  "人工智能的未来发展",
  ["agent_scientist", "agent_engineer", "agent_writer"],
  3  // 3轮对话
);

// 打印所有回复
records.forEach(record => {
  console.log(`${record.agent_name}: ${record.response}`);
});
```

### 11.3 动态添加角色

```typescript
// 创建自定义角色
const customAgent = createBaseAgent(
  "张三",
  "音乐家",
  "agent_musician",
  {
    age: 30,
    personality: "热情开朗",
    background: "专业钢琴演奏家"
  }
);

// 添加到小镇
town.addAgent(customAgent);

// 与新角色对话
const result = await town.chat("你会弹什么曲子？", "agent_musician");
```
