## 一、什么是图工作流引擎？（小白通俗解释）

### 1.1 生活化类比

想象一个**小镇的管理中心**：

```
用户（访客）
    ↓ "我想找医生聊聊"
┌─────────────────────────────┐
│       小镇管理中心            │  ← TownOrchestrator
│                             │
│  ┌──────┐  ┌──────┐  ┌──────┐ │
│  │ 作家  │  │ 医生  │  │ 程序员│ │  ← 各个角色 (Agent)
│  └──────┘  └──────┘  └──────┘ │
│                             │
│  ┌─────────────────────────┐   │
│  │   每个角色的"秘书"     │   │ ← AgentGraph
│  │   (处理对话流程)        │   │
│  └─────────────────────────┘   │
└─────────────────────────────┘
```

**图工作流引擎就是这个"管理中心"+"秘书系统"**，负责：
- 管理镇上所有角色
- 安排用户与角色对话
- 协调记忆、技能、情感等子系统

### 1.2 在赛博小镇中的作用

| 职责 | 说明 |
|------|------|
| 角色管理 | 添加/删除角色，维护角色列表 |
| 对话调度 | 接收用户消息，分发给正确角色 |
| 子系统集成 | 在对话过程中调用记忆、技能、MCP等 |
| 多角色协作 | 支持多角色群聊功能 |

## 三、核心数据结构

### 3.1 TownState（小镇状态）

**文件**: `town_graph.ts:44-51`

```typescript
export interface TownState {
  user_input: string;           // 用户说了什么
  target_agent_id: string;      // 想找谁（用户指定的）

  agent_response: string;       // 角色的回复
  conversation_history: BaseMessage[]; // 对话历史记录

}
```

**类比**：像一张**接待表单**，记录每次来访的信息。

### 3.2 AgentState（角色状态） ⭐ 核心状态

**文件**: `agent_graph.ts:10-20`

```typescript
export interface AgentState {
  messages: BaseMessage[];      // 消息历史
  user_input: string;           // 用户输入
  agent_response: string;       // 角色回复
  memory_context: string;       // ★ 从记忆系统检索到的相关记忆
  emotion_context: string;      // ★ 与对话对象的关系上下文
  matched_skills: string[];     // ★ 匹配到的技能名称列表
  should_continue: boolean;     // 是否继续处理
  agent_id?: string;            // 角色ID
  agent_name?: string;          // 角色名称
}
```

**关键区别**：`TownState`是整个小镇的状态，`AgentState`是单个角色在处理一条消息时的内部状态。

#### `createDefaultTown()` 内部逻辑

**文件**: `town_graph.ts:501-528`

```typescript
async function createDefaultTown(numAgents = 4): Promise<TownOrchestrator> {
  const orchestrator = new TownOrchestrator("赛博小镇");
  
  // 从 DEFAULT_PROFILES 中选择前 N 个职业配置
  const profileKeys = Object.keys(DEFAULT_PROFILES);  // 8种职业
  const selectedProfiles = profileKeys.slice(0, numAgents);
  
  for (const key of selectedProfiles) {
    const profile = DEFAULT_PROFILES[key];  // 预设的角色档案
    
    // 创建角色：name, profession, agentId, profile详情
    const agent = createBaseAgent(
      profile.name,        // 如："张三"
      profile.profession,  // 如："writer"
      `agent_${key}`,     // 如："agent_writer"
      { age, personality, background }
    );
    
    // 设置额外的属性（爱好、说话风格、外貌）
    (agent as any).profile.hobbies = profile.hobbies;
    (agent as any).profile.speech_style = profile.speech_style;
    
    orchestrator.addAgent(agent);
  }
  
  return orchestrator;
}
```

**8种默认职业**: writer(作家), doctor(医生), coder(程序员), teacher(教师), artist(艺术家), engineer(工程师), scientist(科学家), businessman(商人)

## 五、AgentGraph（角色对话引擎）⭐⭐ 核心

### 5.1 是什么？

每个角色都有自己的**"私人秘书"**（AgentGraph），专门处理这个角色的所有对话事务。

**文件**: `agent_graph.ts:56`

```typescript
export class AgentGraph {
  private agent: any;               // 角色本身
  private memory: DualMemorySystem;  // 角色的记忆系统
  private skills: SkillRegistry;     // 角色的技能
  private emotion: EmotionEngine;    // 情感引擎
  private otherAgentId: string;      // 对话对方是谁
  
  private state: AgentState;         // 当前处理状态
}
```

### 5.2 processMessage() — 消息处理的完整流程 ⭐ 最重要

**文件**: `agent_graph.ts:622-716`

这是整个系统的**核心方法**！当用户发送消息给某个角色时，这个方法会被调用：

┌─────────────────────────────────────────────────────────────┐
│  Step 9: invokeLlmNode (调用AI模型) ⭐                        │
│  ├─ 组装完整prompt: systemPrompt + 记忆 + 情感 + 技能增强      │

│  ├─ 调用 DeepSeek API                                    │

### 5.3 invokeLlmNode 的细节（Step 9展开）

这是最复杂的步骤，值得单独讲解：

**组装系统提示词的顺序**：

```typescript
let systemPrompt = agent.getSystemPrompt();     // ① 角色基础设定
if (memory_context) systemPrompt += memory_context;  // ② 记忆上下文
if (emotion_context) systemPrompt += emotion_context;  // ③ 情感上下文
for (skill of matched_skills) {
  systemPrompt += skill.getSystemPrompt();        // ④ 技能专业增强
}
```

**预防性压缩机制**（防止token超限）：

```typescript
import { decideCompactionRoute } from '../memory/compression/preemptive_compaction';

// 决定压缩策略
const routeInfo = decideCompactionRoute(messages, systemPrompt, userInput, 8000, 4096);

switch (routeInfo.route) {
  case TRUNCATE_TOOL_RESULTS_ONLY:  // 只截断工具返回结果
    processedMessages = truncateAggregateToolResults(processedMessages, 8000);
    break;
  case COMPACT_THEN_TRUNCATE:      // 先截断再压缩
    // ...
}

if (routeInfo.shouldCompact) {
  // 主动压缩旧消息
  result = await activeCompact(oldMessages, 8000, llmModel);
}
```

**工具调用循环**（如果LLM决定调用外部工具）：

```typescript
// 1. LLM返回: "我要搜索一下" + tool_calls: [{function: "web_search", arguments: {...}}]

// 2. 执行MCP工具
for (const toolCall of response.tool_calls) {
  const result = await client.callTool(toolCall.function.name, args);
  toolResults.push(result);
}

// 3. 将工具结果发给LLM，让它生成最终回复
const finalResponse = await llmModel.invoke([...messages, ...toolResults]);
state.agent_response = finalResponse.content;
```

## 六、节点系统详解

### 6.1 所有节点一览

| 节点文件 | 功能 | 调用时机 |
|----------|------|---------|
| `middlewarePreCheckNode` | 安全检查（Guardrail+Loop+Summarization） | LLM调用前 |

### 6.2 中间件节点详解（3个）

**文件**: `nodes/middleware_nodes.ts`

#### middlewarePreCheckNode — 前置检查

```typescript
// 在 LLM 调用之前执行
// 注册了3个中间件: GuardrailMiddleware, LoopDetectionMiddleware, MemorySummarizationMiddleware

const context: MiddlewareContext = {
  phase: MiddlewarePhase.BEFORE_THINK,  // 思考前阶段
  goal: state.user_input,
  metadata: { matched_skills, memory_context }
};

const result = manager.process(context, null);
if (!result.should_continue) {
  state.agent_response = result.message || '[安全限制] 操作被阻止';
  state.should_continue = false;
}
```

#### middlewareToolCheckNode — 工具检查

在技能处理之后、LLM调用之前执行。同样的中间件管道，但阶段不同（`BEFORE_ACTION`）。

#### middlewarePostCheckNode — 后置检查

在LLM调用之后执行。可以**修改输出内容**或**阻止结果返回**。

## 九、常见问题FAQ

### Q1: TownOrchestrator 和 AgentGraph 的区别？
**A**: 
- **TownOrchestrator** = 小镇的**管理层**（市长办公室），管理所有角色
- **AgentGraph** = 每个角色的**私人秘书**，只处理一个角色的对话

一个 Orchestrator 管理多个 AgentGraph。

### Q2: 添加新的默认角色怎么做？
**A**: 修改 `agents/models.ts` 中的 `DEFAULT_PROFILES`，添加新的职业配置即可。

### Q3: 对话历史存在哪里？
**A**: 
- **短期**: 内存中的 `state.messages`
- **会话级**: 通过 `RoleHistoryManager` 保存到文件
- **长期**: ChromaDB 向量数据库（语义搜索用）

### Q4: 如何调整对话流程？
**A**: 修改 `agent_graph.ts` 的 `processMessage()` 方法，增删或重排步骤。
