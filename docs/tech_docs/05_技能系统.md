# 赛博小镇 V2 - 技能系统技术文档

## 1. 概述

技能系统为每个职业Agent提供专业化的能力扩展。通过插件化的Skill架构，Agent可以获得特定领域的专业知识和技能，如作家的创意写作、医生的医疗咨询、程序员的代码审查等。

## 2. 架构设计

### 2.1 核心组件

```
SkillRegistry (技能注册中心)
├── Skill (技能基类)
│   ├── WriterSkills (作家技能)
│   ├── DoctorSkills (医生技能)
│   ├── CoderSkills (程序员技能)
│   └── TeacherSkills (教师技能)
├── SkillManifest (技能清单)
└── MCP Dependencies (MCP依赖)
```

### 2.2 技能匹配流程

```
用户输入 → 关键词匹配 → 意图识别 → 选择最佳Skill 
→ 加载MCP依赖 → 执行Skill → 返回结果
```

## 3. 核心类详解

### 3.1 BaseSkill (技能基类)

所有技能的抽象基类：

```typescript
abstract class BaseSkill {
  manifest: SkillManifest;
  
  abstract execute(context: any): Promise<string>;
  getPromptEnhancement(): string;
  getMCPRequirements(): MCPDependency[];
  shouldTrigger(userInput: string, detectedIntent?: string): boolean;
}
```

### 3.2 SkillManifest (技能清单)

声明式配置定义技能属性：

```typescript
interface SkillManifest {
  name: string;                    // 技能名称
  description: string;             // 技能描述
  version: string;                 // 版本号
  triggerKeywords: string[];       // 触发关键词
  triggerIntent: string;           // 意图匹配
  mcpDependencies: MCPDependency[]; // MCP依赖
  systemPromptEnhancement: string; // 提示词增强
  priority: number;                // 优先级
}
```

### 3.3 MCPDependency (MCP依赖)

声明技能所需的外部工具：

```typescript
interface MCPDependency {
  serverName: string;              // MCP服务器名称
  serverConfigPath: string;        // 配置文件路径
  requiredTools: string[];         // 需要的工具列表
  description: string;             // 依赖说明
}
```

### 3.4 SkillRegistry (技能注册中心)

全局技能管理器：

```typescript
class SkillRegistry {
  private skills: Record<string, Record<string, BaseSkill>>;
  
  register(occupation: string, skill: BaseSkill): void;
  matchSkill(occupation: string, userInput: string): BaseSkill | null;
  getAllSkills(occupation: string): BaseSkill[];
  findMatchingSkills(input: string): any[];
  getSkill(name: string): any;
  cleanupAll(): Promise<void>;
}
```

## 5. 技能匹配算法

### 5.1 匹配策略

1. **关键词匹配**: 检查用户输入是否包含触发关键词
2. **意图匹配**: 使用LLM识别用户意图
3. **优先级排序**: 按priority值排序，选择最优技能

### 5.2 匹配代码

```typescript
shouldTrigger(userInput: string, detectedIntent?: string): boolean {
  // 关键词匹配
  const keywordMatch = this.manifest.triggerKeywords.some(
    kw => userInput.includes(kw)
  );
  
  // 意图匹配
  const intentMatch = detectedIntent && 
    this.manifest.triggerIntent.includes(detectedIntent);
  
  return keywordMatch || intentMatch;
}
```

## 6. MCP懒加载机制

### 6.1 设计原理

Skill中声明MCP依赖，只有当Skill被激活时才加载对应的MCP Server。

### 6.2 加载流程

```
Skill匹配成功 → 检查MCP依赖 → 未加载则懒加载 
→ 转换MCP tools为LangChain tools → 绑定到LLM
```

### 6.3 优势

- **节省资源**: 避免启动时加载所有MCP Server
- **按需加载**: 只在需要时加载相关工具
- **声明式配置**: 清晰可控的依赖管理

## 7. 技能执行

### 7.1 执行上下文

```typescript
interface SkillContext {
  userInput: string;              // 用户输入
  conversationHistory: BaseMessage[]; // 对话历史
  agentProfile: AgentProfile;     // Agent档案
  memory: DualMemorySystem;       // 记忆系统

```

### 7.2 执行流程

```typescript
async execute(context: SkillContext): Promise<string> {
  // 1. 获取增强提示词
  const promptEnhancement = this.getPromptEnhancement();
  
  // 2. 准备工具
  const tools = await this.loadMCPTools();
  
  // 3. 构建完整提示
  const fullPrompt = this.buildPrompt(context, promptEnhancement);
  
  // 4. 调用LLM
  const response = await this.llm.invoke(fullPrompt, { tools });
  
  // 5. 返回结果
  return response.content;
}
```

```typescript
      priority: 10
    });
  }

    // 自定义执行逻辑
    return "技能执行结果";
  }
}

// 注册技能
skillRegistry.register("writer", new CustomSkill());
```

### 9.2 技能组合

多个技能可以组合使用，形成更强大的能力：

```typescript
class CompositeSkill extends BaseSkill {
  private subSkills: BaseSkill[];
  
  async execute(context: any): Promise<string> {
    let result = "";
    for (const skill of this.subSkills) {
      result += await skill.execute(context);
    }
    return result;
  }
}
```

## 10. 性能优化

### 10.1 缓存策略

- **技能匹配缓存**: 缓存常用输入的匹配结果
- **MCP连接缓存**: 复用已建立的MCP连接
- **提示词缓存**: 缓存生成的提示词模板

### 10.2 懒加载

- **延迟初始化**: 首次使用时才初始化技能
- **按需加载MCP**: 仅在需要时加载MCP依赖
- **异步加载**: 非阻塞地加载技能资源
