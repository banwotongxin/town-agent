# 赛博小镇 V2 - 记忆管理系统技术文档

## 1. 概述

记忆管理系统是赛博小镇V2的核心组件之一，负责管理智能体的短期记忆和长期记忆。系统采用双层记忆架构，结合滑动窗口机制和向量数据库，实现了高效的记忆存储、检索和压缩功能。

## 2. 架构设计

### 2.1 双层记忆架构

```
DualMemorySystem (双记忆系统)
├── ShortTermMemory (短期记忆)
│   ├── 滑动窗口机制
│   ├── 对话摘要
│   └── 工具结果LRU缓存
├── LongTermMemory (长期记忆)
│   ├── ChromaDB实现
│   ├── PostgreSQL实现
│   └── 模拟实现
└── SessionMemory (会话记忆)
    ├── 历史对话摘要
    └── 记忆提取机制
```

### 2.2 核心组件

- **ShortTermMemory**: 管理当前对话的短期记忆，使用滑动窗口机制
- **LongTermMemory**: 持久化存储重要信息，支持多种后端实现
- **SessionMemory**: 管理会话级别的历史对话摘要
- **DualMemorySystem**: 协调短期和长期记忆的交互

## 3. 核心类详解

### 3.1 MemoryItem接口

定义了记忆项的基本结构：

```typescript
interface MemoryItem {
  id: string;                    // 记忆唯一标识符
  content: string;               // 记忆内容
  importance: number;            // 重要性评分 (0-1)
  timestamp: number;             // 时间戳
  metadata: Record<string, any>; // 元数据
  toDict(): Record<string, any>; // 转换为字典
}
```

### 3.2 ShortTermMemory类

短期记忆管理器，负责维护当前对话上下文。

#### 3.2.1 主要特性

- **滑动窗口机制**: 默认保留最近5轮对话
- **自动压缩**: 当消息数量超过阈值时自动压缩旧对话
- **工具结果LRU缓存**: 最多保留5个最新的工具执行结果
- **Token计数**: 实时监控token使用情况

#### 3.2.2 核心方法

##### addMessage()
添加消息到短期记忆，自动更新对话轮数和工具结果缓存。

##### getMessages()
获取当前窗口的消息列表，如果token超限则触发压缩。

##### getContext()
生成完整的上下文字符串，包含摘要和所有消息。

##### compress()
将旧对话压缩为摘要，减少token占用。

### 3.3 LongTermMemory类

长期记忆存储器，提供持久化的记忆存储和检索功能。

#### 3.3.1 支持的存储后端

1. **ChromaDB**: 基于向量数据库的语义检索（推荐）
2. **PostgreSQL**: 关系型数据库存储
3. **Mock**: 内存模拟存储（用于测试）

#### 3.3.2 核心方法

##### addMemory()
添加新的记忆项，支持设置重要性和元数据。

##### search()
根据查询字符串搜索相关记忆，返回按重要性排序的结果。

##### getAllMemories()
获取所有存储的记忆项。

### 3.4 DualMemorySystem类

双记忆系统的核心协调器，整合短期和长期记忆。

#### 3.4.1 初始化

```typescript
constructor(
  agentId: string,
  shortTermWindow: number = 5,
  longTermStoragePath: string = "./chroma"
)
```

#### 3.4.2 核心方法

##### addMessage()
添加消息到短期记忆，并检查是否需要提取会话记忆。

##### getContext()
获取完整的上下文，包括：
1. 短期记忆（当前窗口）
2. 会话记忆（历史对话摘要）
3. 相关知识库（基于查询的长期记忆）

##### saveImportantEvent()
保存重要事件到长期记忆。

##### compactSessionMemory()
压缩会话记忆，清空短期记忆并保留摘要。

## 4. 记忆提取机制

### 4.1 SessionMemory类

负责从对话中提取重要信息并生成会话记忆摘要。

#### 4.1.1 提取策略

- **基于Token阈值**: 当对话token数超过设定值时触发提取
- **基于对话轮数**: 每N轮对话后触发提取
- **重要性评估**: 使用LLM评估对话的重要性

#### 4.1.2 记忆格式

生成的会话记忆包含：
- 对话主题摘要
- 关键信息点
- 用户偏好和需求
- 待跟进事项

### 4.2 记忆提取流程

```
1. 检测提取条件 → 2. 收集对话内容 → 3. LLM生成摘要 
→ 4. 保存到SessionMemory → 5. 清空短期记忆
```

## 5. Token管理

### 5.1 TokenUtils工具类

提供token计算和管理功能：

- `calculateMessagesTokenCount()`: 计算消息列表的token总数
- `truncateMessagesToTokenLimit()`: 截断消息以符合token限制
- `estimateTokens()`: 估算文本的token数量

### 5.2 Token限制策略

- **短期记忆窗口**: 默认5轮对话，约5000 tokens
- **压缩阈值**: 当token数超过1.5倍窗口大小时触发压缩
- **最大上下文**: 控制在8000 tokens以内

## 6. 工具结果管理

### 6.1 LRU缓存机制

短期记忆中使用LRU（Least Recently Used）缓存管理工具执行结果：

- **最大容量**: 默认保留5个最新的工具结果
- **自动清理**: 超出容量时删除最旧的结果
- **占位符替换**: 被清除的工具结果替换为占位符文本

### 6.2 工具结果格式

```typescript
class ToolResultMessage implements BaseMessage {
  content: string;      // 工具执行结果
  metadata: {
    tool_name: string;  // 工具名称
    status: string;     // 执行状态
    // ...其他元数据
  }
}
```

- **元数据过滤**: 支持按重要性、时间等条件过滤

### 7.2 PostgreSQL集成

提供PostgreSQL作为替代的长期记忆存储方案：

- **表格结构**: memories表存储记忆项
- **索引优化**: 对重要字段建立索引
- **事务支持**: 保证数据一致性

### 7.3 文件存储

短期记忆和会话记忆使用JSON文件存储：

- **路径结构**: `memory_storage/roles/{agent_id}/`
- **文件格式**: history.json, compressed.json
- **备份机制**: 定期创建备份文件

## 8. 记忆压缩算法

### 8.1 滑动窗口压缩

当对话超过窗口大小时，自动压缩旧对话：

```typescript
private compress(messagesToCompress: BaseMessage[]): void {
  const compressedText = messagesToCompress.map(msg => {
    return `${msg.type}: ${msg.content}`;
  }).join('\n');

  if (this.summary) {
    this.summary += `\n[旧对话]\n${compressedText}`;
  } else {
    this.summary = `[历史对话摘要]\n${compressedText}`;
  }

  this.messages = this.messages.slice(messagesToCompress.length);
}
```

### 8.2 LLM辅助压缩

使用LLM生成更智能的对话摘要：

- **主动压缩**: 在特定条件下触发LLM压缩
- **抢占式压缩**: 预测性地压缩可能超标的对话
- **质量审计**: 验证压缩后的摘要质量

### 9.2 基本操作

## 10. 性能优化

### 10.1 缓存策略

- **LRU缓存**: 工具结果使用LRU缓存
- **内存缓存**: 频繁访问的记忆项缓存在内存中
- **批量操作**: 批量写入减少I/O次数

### 10.2 异步处理

所有I/O操作都使用async/await模式：

- 非阻塞的文件读写
- 并发的数据库操作
- 高效的网络请求

### 10.3 资源管理

- **连接池**: 数据库连接复用
- **内存控制**: 限制短期记忆大小
- **垃圾回收**: 定期清理过期数据

## 11. 错误处理

### 11.1 异常处理

- **文件操作失败**: 记录日志并使用fallback机制
- **数据库连接失败**: 重试机制和降级策略
- **LLM调用失败**: 使用默认摘要或跳过压缩

### 11.2 数据完整性

- **事务支持**: 保证原子性操作
- **数据验证**: 验证记忆项的格式和内容
- **备份恢复**: 定期备份支持数据恢复
