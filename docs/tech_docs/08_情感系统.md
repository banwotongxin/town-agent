## 一、什么是情感系统？（小白通俗解释）

### 1.1 生活化类比

在赛博小镇中，角色不是冷冰冰的程序，他们之间有**"关系亲疏"**：

```
刚认识时: "您好，请问有什么可以帮您？"     ← 陌生，客气
熟悉之后: "嗨！今天怎么样？"               ← 朋友，自然
关系很好: "老铁！有啥事尽管说！"           ← 挚友，随意
```

**情感系统就是管理这些"关系亲疏"的机制**，让角色之间的对话会根据关系变化而不同。

### 1.2 核心功能

| 功能 | 说明 |
|------|------|
| 关系等级 | 从"陌生人"到"挚友"5个等级 |
| 情感分数 | 用0-100的数字衡量关系深浅 |
| 互动影响 | 每次对话都会改变关系 |
| 时间衰减 | 长时间不联系，关系会变淡 |
| 对话风格 | 不同等级用不同语气说话 |

## 二、情感等级体系

### 2.1 五级关系

```typescript
enum EmotionLevel {
  STRANGER = 1,      // 陌生人 (0-20分)
  ACQUAINTANCE = 2,  // 泛泛之交 (20-40分)
  FRIEND = 3,        // 朋友 (40-60分)
  GOOD_FRIEND = 4,   // 好友 (60-80分)
  CLOSE_FRIEND = 5   // 挚友 (80-100分)
}
```

**图示**：

```
0 ──── 20 ──── 40 ──── 60 ──── 80 ──── 100
│       │        │        │        │
陌生人  泛交    朋友     好友     挚友
(1)    (2)      (3)      (4)      (5)

初始值: 10分（陌生人）
```

### 2.2 各等级的对话风格

系统为每个等级定义了**对话提示词**，指导AI用什么语气说话：

| 等级 | 称呼 | 语气 | 话题深度 |
|------|------|------|----------|
| 陌生人 | 您/先生/女士 | 礼貌、正式 | 浅表、不涉及隐私 |
| 泛交 | 姓+称呼 | 友好但有距离 | 日常话题 |
| 朋友 | 直呼其名 | 轻松自然 | 分享生活感受 |
| 好友 | 昵称/外号 | 亲密不拘束 | 深层次想法 |
| 挚友 | 专属昵称 | 完全坦诚信任 | 无话不谈 |

## 三、核心类详解

### 3.1 RelationshipState（关系状态）

记录两个角色之间的**完整关系信息**：

```typescript
class RelationshipState {
  agentAId: string;           // 角色A的ID
  agentBId: string;           // 角色B的ID
  emotionScore: number;       // 情感分数 (0-100)
  level: EmotionLevel;        // 当前等级
  interactionCount: number;   // 累计互动次数
  lastInteractionTime: number;// 最后互动时间戳
  nickname?: string;          // 特殊昵称（可选）
}
```

**创建示例**：

```typescript
import { RelationshipState, EmotionLevel } from './emotion';

// 创建一段新关系（默认是陌生人）
const rel = new RelationshipState(
  'agent_zhangsan',    // 张三的ID
  'agent_lisi',        // 李四的ID
  10.0,                // 初始分数10（陌生人）
  EmotionLevel.STRANGER,
  0,                   // 互动次数0
  0                    // 最后互动时间
);

// 输出关系信息
console.log(rel.toDict());
// {
//   agent_a_id: "agent_zhangsan",
//   agent_b_id: "agent_lisi",
//   emotion_score: 10,
//   level: 1,
//   level_name: "陌生人",
//   interaction_count: 0
// }
```

### 3.2 EmotionCalculator（情感计算器）⭐ 核心

负责计算每次互动后**情感分数的变化量**。

#### 参数说明

```typescript
const calculator = new EmotionCalculator(
  baseIncrease: 3.0,     // 正向互动基础加分
  baseDecrease: 5.0,     // 负向互动基础扣分（惩罚更重！）
  decayRate: 0.5,        // 每天衰减量（长时间不联系）
  noInteractionDays: 7   // 多少天后开始衰减
);
```

#### 计算公式

```
最终变化量 = 基础变化 × 类型系数 × 等级系数

其中:
- 基础变化: positive=+3, negative=-5, neutral=+0.5
- 类型系数: conversation=1.0, help=1.5, gift=1.3, conflict=-2.0...
- 等级系数: 等级越高提升越难 (最高0.4倍)
```

#### 使用示例

```typescript
// 计算一次正向对话的变化量
const delta = calculator.calculateDelta(
  'conversation',        // 互动类型：普通对话
  25.0,                  // 当前分数：25（泛交级别）
  EmotionLevel.ACQUAINTANCE, // 当前等级：泛交
  'positive'             // 情感倾向：正面
);
console.log(delta);  // → 大约 +2.55 分
```

#### 互动类型系数表

| 类型 | 含义 | 系数 |
|------|------|------|
| `conversation` | 普通对话 | ×1.0 |
| `help` | 帮助行为 | ×1.5 |
| `gift` | 赠送礼物 | ×1.3 |
| `conflict` | 冲突争执 | ×(-2.0) |
| `praise` | 表扬赞美 | ×1.2 |
| `criticism` | 批评指责 | ×(-1.5) |

#### 时间衰减

```typescript
// 如果7天没联系，每天扣0.5分
const newScore = calculator.applyTimeDecay(currentScore, 15); // 15天没联系
// newScore = currentScore - (15-7) × 0.5 = currentScore - 4
```

### 3.3 RelationshipStore（关系存储）

管理所有关系的**增删查**：

```typescript
const store = new RelationshipStore();

// 获取关系
let rel = store.getRelationship('agent_a', 'agent_b');

// 不存在则自动创建（默认10分）
rel = store.getOrCreate('agent_a', 'agent_b', 10);

// 手动设置关系
store.setRelationship('agent_a', 'agent_b', newRel);

// 获取某个角色的所有关系
const allRels = store.getAllRelationships('agent_a');
```

**关键实现细节**：

```typescript
// 关系键生成规则：两个ID排序后连接
// 保证 A→B 和 B→A 是同一个关系
private makeKey(agentAId: string, agentBId: string): string {
  return [agentAId, agentBId].sort().join("-");
}
// makeKey('zhangsan', 'lisi') → "lisi-zhangsan"
// makeKey('lisi', 'zhangsan') → "lisi-zhangsan" （相同！）
```

### 3.4 EmotionEngine（情感引擎）⭐⭐ 主入口

整合以上所有组件的**顶层管理者**：

```typescript
import { getEmotionEngine } from './emotion';

const engine = getEmotionEngine(); // 全局单例
```

#### 核心方法：interact()

让两个角色进行一次互动：

```typescript
const result = engine.interact(
  'zhangsan',            // 角色A
  'lisi',                // 角色B
  'conversation',         // 互动类型
  'positive'              // 情感倾向
);

console.log(result);
// {
//   old_score: 25.00,        // 互动前的分数
//   new_score: 27.55,        // 互动后的分数
//   delta: 2.55,             // 变化量
//   old_level: "泛泛之交",   // 旧等级
//   new_level: "泛泛之交",   // 新等级（可能不变）
//   level_changed: false,    // 是否升级
//   event_triggered: false   // 是否触发特殊事件
// }
```

#### 获取关系信息

```typescript
const info = engine.getRelationshipInfo('zhangsan', 'lisi');
// → { agent_a_id: ..., emotion_score: 45, level: 3, level_name: "朋友", ... }
```

#### 获取对话风格提示

```typescript
// 根据等级获取AI应该使用的对话风格
const hint = engine.getConversationStyleHint(EmotionLevel.FRIEND);
console.log(hint);
/*
[对话风格：朋友]
- 可以直接称呼名字
- 语气轻松自然
- 分享日常生活和感受
- 互相关心，提供帮助
- 可以开一些善意的玩笑
*/
```

## 四、完整工作流程

```
用户与角色B对话
      ↓
调用 engine.interact(user, B, 'conversation', 'positive')
      ↓
┌─────────────────────────────────────────────┐
│ Step 1: 获取或创建关系                       │
│   store.getOrCreate(agentA, agentB)          │
│   → 返回 RelationshipState                   │
└─────────────────────┬───────────────────────┘
                      ↓
┌─────────────────────────────────────────────┐
│ Step 2: 计算情感变化                         │
│   calculator.calculateDelta(type,score,level,sentiment) │
│   → 得到 delta 变化量                        │
└─────────────────────┬───────────────────────┘
                      ↓
┌─────────────────────────────────────────────┐
│ Step 3: 更新关系状态                         │
│   score += delta                            │
│   interactionCount += 1                     │
│   lastInteractionTime = now                 │
└─────────────────────┬───────────────────────┘
                      ↓
┌─────────────────────────────────────────────┐
│ Step 4: 检查等级变化                         │
│   relationship.updateLevel()                │
│   → 判断是否需要升级/降级                     │
│   → 升级时可能触发事件                       │
└─────────────────────┬───────────────────────┘
                      ↓
              返回互动结果
```

## 五、实际应用场景

### 场景1：小镇中的角色社交

```typescript
// 张三帮李四解决了问题 → 关系升温
engine.interact('zhangsan', 'lisi', 'help', 'positive');
// score: 30 → 34.5 (+4.5, help有1.5倍加成)

// 后来两人发生了冲突 → 关系下降
```

## 七、设计亮点总结

| 设计点 | 说明 |
|--------|------|
| **边际递减效应** | 关系越好，越难提升（防止刷分） |
| **惩罚大于奖励** | 扣分比加分的绝对值大（负面印象更深） |
| **时间衰减** | 长期不联系会淡忘 |
| **不可跨级升级** | 每次最多提升到当前等级上限的80% |
| **双向关系键** | 排序后拼接ID，保证一致性 |

## 八、常见问题FAQ

### Q1: 为什么正向互动只加3分，负向却扣5分？
**A**: 这是模拟现实心理学——**坏事比好事的影响更大**。一次冲突可能毁掉多次友好互动建立的关系。

### Q2: 分数到100后就不再增加了吗？
**A**: 不是。分数可以超过100（上限由计算逻辑控制），但**等级最高是CLOSE_FRIEND (80-100)**。超过100只是表示关系非常稳固。

### Q3: 如何自定义初始分数？
**A**: 
```typescript
store.getOrCreate('a', 'b', 50);  // 初始50分，直接是朋友关系
```

### Q4: 关系数据会持久化保存吗？
**A**: 当前版本存在内存中，程序重启后丢失。后续可以对接数据库或记忆系统实现持久化。
