## 二、Agent 的基本结构

### 2.1 BaseAgent 基类（所有角色的"祖先"）

```
BaseAgent（基础智能体）
│
├── 属性：
│   ├── agentId: string        ← 身份证号（唯一标识）
│   ├── profile: AgentProfile   ← 个人档案（名字、年龄等）
│   ├── llmModel: LLMModel     ← AI大脑（用来思考回复）
│   ├── isActive: boolean      ← 是否在线
│   ├── currentLocation: string ← 当前位置
│   └── conversationCount: number ← 对话次数
│
├── 方法：
│   ├── getSystemPrompt()    ← 获取自我介绍（我是谁）
│   ├── formatMessages()      ← 组织对话消息
│   └── respond()            ← 回复用户消息（核心功能）
│
└── 继承者：
    ├── SubAgent（子智能体）- 执行具体任务
    ├── LeaderAgent（领导）- 分配任务
    └── VerificationAgent（验证员）- 检查质量
```

### 2.2 代码中的位置

```typescript
// 文件位置：src/AI/agents/base_agent.ts

export class BaseAgent {
  private agentId: string;         // 角色ID
  private profile: AgentProfile;   // 角色档案
  private llmModel?: LLMModel;     // AI大脑
  
  // 核心方法：回复用户
  async respond(userMessage, conversationHistory, kwargs) {
    // 1. 加载历史对话
    // 2. 组装消息
    // 3. 调用AI模型
    // 4. 保存对话记录
    return response;
  }
}
```

```
│     "你好啊！"                                       │

│  🤖 AIMessage（AI回复）                              │
│     "你好！很高兴见到你..."                          │

│     "你是一个作家，请用文艺的语气回复"              │

│                                                     │
│  🔧 ToolMessage（工具调用）                           │

│                                                     │
│  📋 ToolResultMessage（工具结果）                     │

│                                                     │

```

### 3.2 消息的生命周期

```
用户发送 "你好"
    ↓
创建 HumanMessage { content: "你好", type: "human" }
    ↓
系统添加 SystemMessage { content: "你是作家林墨...", type: "system" }
    ↓
AI生成回复 → 创建 AIMessage { content: "你好呀...", type: "ai" }
    ↓
返回给用户展示
```

## 五、LLM 模型接口

### 5.1 什么是 LLM？

**LLM = Large Language Model（大语言模型）**

就是让 AI 能够"理解"和"生成"人类语言的程序。赛博小镇使用的是：

- **DeepSeek**（主要模型）- 国产优秀大模型
- **通义千问**（辅助模型）- 阿里云的大模型

### 5.2 LLM 接口定义

```typescript
// LLM 就像是一个"遥控器"，有两个按钮：

interface LLMModel {
  // 按钮1：发送消息，等待回复
  invoke(
    messages: BaseMessage[],      // 对话历史
    options?: { tools?: any[] }   // 可选：可用工具
  ): Promise<{
    content: string;     // AI的回复
    tool_calls?: any[];   // 是否要调用工具
  }>;
  
  // 按钮2：异步调用（性能更好）
  ainvoke?(...): Promise<...>;
}
```

### 5.3 实际使用示例

```typescript
// 在 web_server.ts 中的实际实现

class DeepSeekModel {
  async invoke(messages, options?) {
    // 1. 格式化消息（转换成DeepSeek需要的格式）
    const formattedMessages = messages.map(msg => ({
      role: msg.type === 'human' ? 'user' : 'assistant',
      content: msg.content
    }));
    
    // 2. 发送API请求到 DeepSeek
    const response = await fetch('https://api.deepseek.com/v1/chat/completions', {
      method: 'POST',
      headers: {
        'Authorization': `Bearer ${this.apiKey}`,
        'Content-Type': 'application/json'
      },
      body: JSON.stringify({
        model: 'deepseek-chat',
        messages: formattedMessages,
        temperature: 0.7,  // 创造性参数（0-1，越高越有创意）
        max_tokens: 2000    // 最大回复长度
      })
    });
    
    // 3. 返回结果
    const data = await response.json();
    return {
      content: data.choices[0].message.content,
      tool_calls: data.choices[0].message.tool_calls
    };
  }
}
```

## 六、Agent 的生命周期

### 6.1 从创建到工作

```
步骤1: 创建实例
  const agent = new BaseAgent("agent_001", profile, llmModel);
  
       ↓
步骤2: 初始化状态
  agent.isActive = true;
  agent.currentLocation = "home";
  agent.conversationCount = 0;
  
       ↓
步骤3: 开始接收用户消息
  while (用户还在聊天) {
    const reply = await agent.respond(userMessage, history);
    显示回复给用户
    agent.conversationCount++;
  }
  
       ↓
步骤4: （可选）停用或销毁
  agent.isActive = false;
```

### 6.2 respond() 方法详解（核心流程）

这是 Agent 最重要的方法！当用户发消息时：

```typescript
async respond(userMessage, conversationHistory, kwargs) {
  
  // ❌ 如果没有大脑（llmModel），无法回复
  if (!this.llmModel) {
    return "[系统] 我还没有学会说话...";
  }
  
  // 📂 第一步：从文件加载历史对话（长期记忆）
  let fileHistory = await roleHistoryManager.getContext(this.agentId);
  
  // 🔄 第二步：合并历史 + 当前消息
  const allHistory = [...fileHistory, ...conversationHistory];
  
  // 📝 第三步：格式化消息（添加系统提示、记忆上下文等）
  const messages = this.formatMessages(
    [...allHistory, new HumanMessage(userMessage)],
    kwargs.memory_context,   // 可选：之前的记忆摘要
    kwargs.emotion_context   // 可选：情感关系状态
  );
  
  // 🧠 第四步：调用 AI 大脑生成回复
  const response = await this.llmModel.invoke(messages);
  
  // 💾 第五步：保存这次对话到文件（持久化）
  await roleHistoryManager.addMessage(this.agentId, 
    new HumanMessage(userMessage));
  await roleHistoryManager.addMessage(this.agentId, 
    new AIMessage(response.content));
  
  // ✅ 第六步：返回回复
  return response.content;
}
```

## 八、常见问题排查

### Q1: 角色不回复？
**检查清单：**
- [ ] `.env` 文件中 API Key 是否正确？
- [ ] 网络是否正常（能访问 DeepSeek API）？
- [ ] 控制台是否有报错信息？

### Q2: 角色回复不符合人设？
**可能原因：**
- System Prompt（系统提示）不够明确
- 对话历史太长，冲淡了人设信息
- **解决方法**: 编辑 `getSystemPrompt()` 方法，强化角色设定

### Q3: 如何让角色记住之前的内容？
**记忆机制已自动处理：**
- 短期记忆：最近 N 轮对话自动保留
- 长期记忆：重要信息存入 ChromaDB
- 文件记忆：所有对话保存到 JSON 文件

### Q4: 如何调试 Agent？
```typescript
// 在代码中添加日志

console.log(`[Agent ${this.profile.name}] 历史记录条数:`, allHistory.length);
```
