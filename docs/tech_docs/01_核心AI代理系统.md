# 赛博小镇 V2 - 核心AI代理系统技术文档

## 1. 概述

核心AI代理系统是赛博小镇V2的核心组件，负责管理所有智能体（Agent）的生命周期、行为和交互。系统基于TypeScript实现，采用面向对象的设计模式，提供了灵活的Agent架构。

## 2. 架构设计

### 2.1 类层次结构

```
BaseAgent (基础代理)
├── SubAgent (子代理)
├── TeamAgent (团队代理)
├── LeaderAgent (领导代理)
├── VerificationAgent (验证代理)
└── MemoryExtractionAgent (记忆提取代理)
```

### 2.2 核心组件

- **BaseAgent**: 所有代理的基类，提供基本的对话和状态管理功能
- **AgentProfile**: 代理的个人档案，包含姓名、年龄、职业等信息
- **消息系统**: HumanMessage, AIMessage, SystemMessage, ToolMessage等消息类型
- **LLM接口**: 与大型语言模型交互的标准接口

## 3. 核心类详解

### 3.1 BaseAgent类

BaseAgent是所有智能体的基类，提供了以下核心功能：

#### 3.1.1 主要属性

- `agentId`: 智能体唯一标识符
- `profile`: 智能体个人档案
- `llmModel`: 语言模型实例
- `isActive`: 激活状态
- `currentLocation`: 当前位置
- `currentMood`: 当前情绪
- `conversationCount`: 对话次数计数

#### 3.1.2 核心方法

##### respond() 方法
处理用户消息并生成回复的核心方法：

```typescript
async respond(
  userMessage: string,
  conversationHistory: BaseMessage[],
  kwargs: Record<string, any> = {}
): Promise<string>
```

工作流程：
1. 检查语言模型可用性
2. 从文件加载历史对话记录
3. 合并历史对话和当前消息
4. 格式化消息（添加系统提示、记忆上下文、情绪上下文）
5. 调用语言模型生成回复
6. 保存对话到文件历史
7. 返回回复内容

##### getSystemPrompt() 方法
生成系统提示信息，包含智能体的基本信息和行为准则。

##### formatMessages() 方法
格式化消息数组，添加系统提示、记忆上下文和情绪上下文。

### 3.2 消息类型系统

系统定义了多种消息类型以支持不同的交互场景：

- **HumanMessage**: 用户发送的消息
- **AIMessage**: AI生成的回复
- **SystemMessage**: 系统指令和配置信息
- **ToolMessage**: 工具调用请求
- **ToolResultMessage**: 工具执行结果

### 3.3 LLM模型接口

定义了与大型语言模型交互的标准接口：

```typescript
interface LLMModel {
  invoke: (messages: BaseMessage[]) => Promise<{ content: string }>;
  ainvoke?: (messages: BaseMessage[]) => Promise<{ content: string }>;
}
```

## 4. 代理类型

### 4.1 SubAgent (子代理)
专门处理特定任务的代理，可以独立运行或作为团队的一部分。

### 4.2 TeamAgent (团队代理)
协调多个子代理协同工作的代理，负责任务分配和结果整合。

### 4.3 LeaderAgent (领导代理)
具有决策能力的代理，能够指导其他代理的工作流程。

### 4.4 VerificationAgent (验证代理)
负责验证其他代理输出质量的代理，确保结果的准确性和一致性。

### 4.5 MemoryExtractionAgent (记忆提取代理)
专门用于从对话中提取重要信息并存储到长期记忆的代理。

## 5. 配置管理

### 5.1 AgentProfile
每个代理都有一个详细的个人档案，包含：

- 基本信息：姓名、年龄、职业
- 性格特征：个性描述
- 背景故事：角色的背景信息
- 兴趣爱好：角色的兴趣列表
- 说话风格：特定的语言表达方式
- 外貌特征：可选的外观描述

### 5.2 工厂函数
提供`createBaseAgent()`工厂函数，简化代理实例的创建过程。

## 6. 持久化机制

### 6.1 对话历史管理
使用RoleHistoryManager管理每个代理的对话历史，支持：

- 从文件系统加载历史对话
- 自动保存新的对话记录
- Token数量控制（默认限制8000 token）
- 错误处理和回退机制

### 6.2 状态管理
每个代理维护自己的状态信息，包括位置、情绪、对话计数等。

## 7. 扩展性设计

### 7.1 继承机制
通过继承BaseAgent类，可以轻松创建具有特定行为的代理类型。

### 7.2 插件化架构
支持通过中间件、技能和工具系统扩展代理功能。

### 7.3 配置驱动
代理行为可以通过配置文件进行定制，无需修改代码。

## 8. 性能优化

### 8.1 异步处理
使用async/await模式处理I/O操作，提高并发性能。

### 8.2 缓存机制
通过文件系统缓存对话历史，减少重复计算。

### 8.3 Token管理
智能控制发送给LLM的token数量，避免超出模型限制。

## 9. 错误处理

系统实现了完善的错误处理机制：

- 语言模型调用失败时的回退策略
- 文件读写错误的容错处理
- 网络异常的处理和重试机制

## 10. 安全考虑

- 输入验证和 sanitization
- 敏感信息保护
- 访问控制机制

## 11. 测试策略

- 单元测试覆盖核心功能
- 集成测试验证组件交互
- 端到端测试确保系统完整性

## 12. 部署和运维

- 支持Docker容器化部署
- 日志记录和监控
- 健康检查机制
