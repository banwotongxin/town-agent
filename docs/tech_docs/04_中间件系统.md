# 赛博小镇 V2 - 中间件系统技术文档

## 1. 概述

中间件系统是赛博小镇V2的安全和稳定性保障层，提供七层防护机制。每个中间件负责特定的检查和修正任务，确保系统安全、稳定、高效运行。

## 2. 架构设计

### 2.1 中间件链

```
请求 → DanglingAction → Guardrail → MemorySummarization 
     → ConcurrentLimit → LoopDetection → Clarification 
     → ErrorHandling → WriteApproval → 响应
```

### 2.2 执行顺序

1. **DanglingActionMiddleware**: 修复悬空操作
2. **GuardrailMiddleware**: 安全策略检查
3. **MemorySummarizationMiddleware**: 记忆摘要
4. **ConcurrentLimitMiddleware**: 并发限制
5. **LoopDetectionMiddleware**: 循环检测
6. **ClarificationMiddleware**: 澄清拦截
7. **ErrorHandlingMiddleware**: 错误处理
8. **WriteApprovalMiddleware**: 写入审批

## 3. 核心中间件详解

### 3.1 BaseMiddleware (基类)

所有中间件的抽象基类，定义了标准接口：

```typescript
abstract class BaseMiddleware {
  abstract name: string;
  abstract priority: number;
  abstract process(state: any): Promise<any>;
}
```

### 3.2 DanglingActionMiddleware

**功能**: 检测和修复悬空的操作调用。

**问题场景**: AI发起工具调用但用户中断对话，导致状态不一致。

**解决方案**: 自动补全缺失的工具结果消息。

### 3.3 GuardrailMiddleware

**功能**: 安全策略检查，阻止危险操作。

**检查项**:
- 禁止执行系统命令（bash）
- 禁止写入敏感文件
- 防止资源滥用

**配置**:
```yaml
guardrails:
  enabled: true
  denied_tools: ["bash", "write_file"]
  fail_closed: true
```

### 3.4 MemorySummarizationMiddleware

**功能**: 上下文压缩，防止token超限。

**触发条件**:
- Token数量超过6000
- 消息数量超过50条

**压缩策略**:
- 保留最近20条消息
- 将旧消息压缩为摘要
- 使用LLM生成智能摘要

### 3.5 ConcurrentLimitMiddleware

**功能**: 限制并发子代理数量，防止资源耗尽。

**默认限制**: 最多3个并发子代理

**处理方式**: 截断超出的子代理调用，添加警告消息。

### 3.6 LoopDetectionMiddleware

**功能**: 检测并打破重复调用的死循环。

**检测策略**:
- 记录工具调用历史
- 滑动窗口大小：20次调用
- 警告阈值：3次重复
- 强制停止：5次重复

### 3.7 ClarificationMiddleware

**功能**: 拦截澄清请求，中断到END节点。

**处理的工具**:
- ask_clarification
- request_clarification
- clarify_input

### 3.8 ErrorHandlingMiddleware

**功能**: 捕获工具执行异常，转换为可读错误消息。

**处理的异常类型**:
- PermissionError: 权限不足
- FileNotFoundError: 文件不存在
- TimeoutError: 执行超时
- NetworkError: 网络错误
- ValueError: 参数错误

### 3.9 WriteApprovalMiddleware

**功能**: 文件写入审批，防止未授权的写操作。

**审批流程**:
1. 检测写文件操作
2. 检查目标路径是否在白名单中
3. 验证文件大小限制
4. 记录写入日志

## 4. 中间件工厂

Factory模式简化中间件创建：

```typescript
function createMiddlewareChain(config: any): Middleware[] {
  return [
    new DanglingActionMiddleware(),
    new GuardrailMiddleware(config.guardrails),
    new MemorySummarizationMiddleware(config.summarization),
    new ConcurrentLimitMiddleware(config.concurrent),
    new LoopDetectionMiddleware(config.loop_detection),
    new ClarificationMiddleware(),
    new ErrorHandlingMiddleware(),
    new WriteApprovalMiddleware(config.write_approval)
  ];
}
```

## 5. 配置管理

### 5.1 YAML配置文件

```yaml
middleware:
  guardrails:
    enabled: true
    fail_closed: true
    denied_tools: ["bash"]
  
  summarization:
    enabled: true
    trigger_tokens: 6000
    trigger_messages: 50
    keep_messages: 20
  
  loop_detection:
    warn_threshold: 3
    hard_limit: 5
    window_size: 20
  
  concurrent_limit:
    max_concurrent: 3
```

### 5.2 运行时配置

支持动态启用/禁用中间件：

```typescript
middleware.setEnabled(false);  // 临时禁用
middleware.updateConfig(newConfig);  // 更新配置
```

## 6. 性能影响

### 6.1 开销分析

| 中间件 | 平均耗时 | 影响程度 |
|--------|---------|---------|
| DanglingAction | <1ms | 低 |
| Guardrail | <5ms | 低 |
| MemorySummarization | 500-2000ms | 高（仅触发时） |
| ConcurrentLimit | <1ms | 低 |
| LoopDetection | <2ms | 低 |
| Clarification | <1ms | 低 |
| ErrorHandling | <1ms | 低 |
| WriteApproval | <5ms | 低 |

### 6.2 优化建议

- 缓存常用检查结果
- 异步执行非关键检查
- 批量处理多个消息

## 7. 扩展开发

### 7.1 创建自定义中间件

```typescript
class CustomMiddleware extends BaseMiddleware {
  name = "custom";
  priority = 50;
  
  async process(state: any): Promise<any> {
    // 自定义逻辑
    return state;
  }
}
```

### 7.2 注册中间件

```typescript
const chain = createMiddlewareChain(config);
chain.push(new CustomMiddleware());
```

## 8. 最佳实践

- 中间件应保持无状态
- 避免在中间件中执行耗时操作
- 详细记录日志便于调试
- 合理设置优先级
- 提供配置开关便于调试

## 9. 故障排查

### 9.1 常见问题

**问题**: 中间件执行失败  
**解决**: 检查日志，确认配置正确

**问题**: 性能下降  
**解决**: 检查是否频繁触发压缩中间件

**问题**: 误拦截正常操作  
**解决**: 调整guardrail配置或白名单

## 10. 监控指标

- 中间件执行次数
- 平均执行时间
- 拦截/修正次数
- 错误率
