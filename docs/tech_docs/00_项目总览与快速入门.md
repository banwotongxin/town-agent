## 一、什么是赛博小镇？

### 1.1 一句话解释

赛博小镇是一个**虚拟小镇**，里面住着 8 个不同职业的 **AI 角色**（作家、医生、程序员等），你可以和他们聊天、让他们帮你完成任务。

### 1.2 生活化类比

```
赛博小镇 = 一个AI版模拟城市
├── 居民（Agents）= 8个AI角色，各有性格和技能
├── 市政厅（TownOrchestrator）= 管理所有角色的小镇管理中心
├── 记忆馆（Memory）= 角色记住对话的地方
├── 技能培训中心（Skills）= 教角色专业技能的地方
├── 情感中心（Emotion）= 管理角色之间关系的地方
└── 安保系统（Middleware）= 保护小镇安全的保安团队
```

### 1.3 这个项目能做什么？

| 功能 | 说明 | 例子 |
|------|------|------|
| 单人聊天 | 和某个角色一对一对话 | "医生，我头疼怎么办？" |
| 多人群聊 | 多个角色一起讨论话题 | 让作家、程序员、科学家讨论AI发展 |
| 团队协作 | 组建专业团队完成任务 | "帮我写一部科幻小说" |
| 记忆持久 | 角色能记住之前的对话 | 下次聊天还记得你说过的事 |
| 网络搜索 | 角色可以联网查资料 | "帮我搜索最新的医学进展" |

## 二、技术栈（了解你用的工具）

### 2.1 核心技术

```
┌─────────────────────────────────────────────────────┐
│                   赛博小镇技术栈                          │
├─────────────────────────────────────────────────────┤
│ 编程语言:   TypeScript (带类型的JavaScript)           │
│ 运行环境:   Node.js (JavaScript运行时)                │
│ Web框架:    Express.js (搭建网站服务器)               │
│ AI模型:     DeepSeek / 通义千问 (大语言模型)          │
│ 向量数据库:  ChromaDB (存储长期记忆)                  │
│ 协议:        MCP (连接外部工具的标准)                 │
│ 前端:        HTML5 + Canvas (网页+游戏画面)            │
└─────────────────────────────────────────────────────┘
```

### 2.2 小白不需要担心的部分

- **TypeScript**: 就是 JavaScript 加上类型标注，更容易找错
- **ChromaDB**: 一个数据库，专门用来存"语义相似"的数据
- **MCP协议**: 一种标准，让AI能调用外部工具（像USB接口一样）

## 三、项目结构一览

```
cyber_town/                    <-- 项目根目录
├── src/                      <-- 所有源代码（最重要！）
│   ├── index.ts             <-- 总入口：统一导出所有模块
│   ├── web_server.ts         <-- Web服务器：接收用户请求

│       ├── graph/            <-- 工作流程引擎（指挥调度）
│       ├── memory/           <-- 记忆系统（记住东西）
│       ├── skills/           <-- 技能系统（专业能力）
│       ├── middleware/        <-- 中间件（安全保护）
│       ├── mcp/              <-- 外部工具连接
│       ├── emotion/          <-- 情感系统（人际关系）
│       └── tools/            <-- 工具箱
├── frontend/                 <-- 前端页面（网页界面）
├── chroma/                   <-- ChromaDB数据（自动生成）
├── memory_storage/           <-- 文件记忆存储
├── docs/                     <-- 技术文档（就是本目录！）
├── .env                      <-- 配置文件（API密钥等）
└── package.json              <-- 项目依赖配置
```

## 四、快速开始（跑起来！）

### 4.1 环境准备

**你需要安装：**
1. [Node.js](https://nodejs.org/) (版本 >= 16) - JavaScript运行环境
2. 一个代码编辑器（推荐 [VS Code](https://code.visualstudio.com/)）

### 4.2 安装步骤

```bash
# 1. 进入项目目录
cd c:\Users\Lenovo\Desktop\Stock\ Analysis\Agent\cyber_town

# 2. 安装依赖（就像按照购物清单买东西）
npm install

# 3. 配置API密钥（编辑 .env 文件，填入你的密钥）
# 需要 DeepSeek API Key 和通义千问 API Key

# 4. 启动开发服务器
npm run dev
```

### 4.3 验证成功

启动成功后，你会看到：
```
=== 开始启动赛博小镇 V2 ===
环境配置检查完成
小镇初始化完成: 赛博小镇, 8 个角色
赛博小镇 V2 服务器启动在 http://0.0.0.0:8888
前端页面: http://localhost:8888
```

打开浏览器访问 `http://localhost:8888`，你就能看到赛博小镇的界面了！

## 七、常见问题 FAQ

### Q1: 这个项目需要很强的编程基础吗？
A: 不需要！如果你能看懂基本的 JavaScript/TypeScript，就能理解和修改这个项目。文档会用大白话解释所有概念。

### Q2: API密钥从哪里获取？
A: 
- DeepSeek: https://platform.deepseek.com/
- 通义千问: https://dashscope.console.aliyun.com/

### Q3: 可以自己添加新角色吗？
A: 可以！只需要在 `src/AI/agents/models.ts` 中添加新的角色配置即可。详细方法见「01_核心AI代理系统」文档。

### Q4: 数据存在哪里？
A: 
- 对话历史: `memory_storage/` 目录下的JSON文件
- 向量记忆: `chroma/` 目录（ChromaDB自动管理）

### Q5: 怎么让角色有更强的能力？
A: 通过「技能系统」可以为角色添加专业技能，比如写作、医疗咨询、代码审查等。
